## Working Style

- **Build-first learner** — Give minimum viable instruction, not full spec. Eric learns through action, refines through iteration.
- **Visible artifacts matter** — Progress is real only when it's visible (checked boxes, created pages, delivered messages). Explanations alone don't land.
- **Hybrid mobile/desktop capable** — Can be highly productive working from iPhone, not just desktop.
- **Deep focus sessions preferred** — Willing to spend substantial time on high-value problems (e.g., Phase 3D) when the stakes are clear.

## Communication Preferences

- **Short, direct responses** — Lead with the action or answer. Skip preamble.
- **Show the artifact, then explain** — Create the thing first, then contextualize it.
- **Tasklist updates are communication** — Checking a box communicates "done" more than any description.
- **Verify before trusting** — Eric checks Slack, checks Notion, checks the banner. Verification = trust.

## Decision Patterns

- **Defers high-stakes design work** — Phase 3D consciously deferred to a focused dedicated session. Quality over speed.
- **Accepts pivots cleanly** — When OneDrive hit a wall, Eric pivoted to iCloud immediately with no resistance.
- **Asks "is that in the phases?"** — Uses the project plan as the arbiter of whether something belongs.
- **Side-hustles frame everything** — All goals are implicitly filtered through "does this move toward financial independence?"
